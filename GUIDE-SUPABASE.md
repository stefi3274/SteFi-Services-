# Guide Supabase — Réalisations & Admin

Ce guide explique comment activer la page **Réalisations** (publique) et la page
**Admin** (privée) pour ajouter/supprimer vos travaux en ligne. Compter ~10 min.
Le plan gratuit de Supabase suffit largement.

---

## Étape 1 — Créer un projet Supabase

1. Aller sur https://supabase.com et créer un compte (gratuit).
2. Cliquer **New project**.
3. Choisir un nom (ex. `stefi-services`), un mot de passe de base de données
   (notez-le quelque part), et une région proche (ex. *East US*).
4. Attendre 1–2 min que le projet se crée.

---

## Étape 2 — Créer la table des réalisations

1. Dans le menu de gauche : **SQL Editor** → **New query**.
2. Coller le code ci-dessous et cliquer **Run** :

```sql
-- Table des réalisations
create table realisations (
  id          bigint generated by default as identity primary key,
  titre       text not null,
  lien        text,
  description text,
  image_url   text,
  created_at  timestamptz default now()
);

-- Activer la sécurité au niveau des lignes
alter table realisations enable row level security;

-- Tout le monde peut LIRE les réalisations (affichage public)
create policy "Lecture publique"
  on realisations for select
  to anon, authenticated
  using (true);

-- Seuls les utilisateurs CONNECTÉS peuvent ajouter / modifier / supprimer
create policy "Ecriture connectee - insert"
  on realisations for insert
  to authenticated
  with check (true);

create policy "Ecriture connectee - update"
  on realisations for update
  to authenticated using (true) with check (true);

create policy "Ecriture connectee - delete"
  on realisations for delete
  to authenticated using (true);
```

> Ces règles sont l'élément de sécurité essentiel : même si le code de la page
> admin est public, personne ne peut ajouter ou supprimer sans être connecté.

---

## Étape 3 — Créer votre compte administrateur

1. Menu de gauche : **Authentication** → **Users** → **Add user** → **Create new user**.
2. Saisir votre **email** et un **mot de passe** solides. Cocher
   *Auto Confirm User* si proposé (pour ne pas avoir à valider par email).
3. C'est ce couple email + mot de passe que vous utiliserez sur la page
   `admin.html` pour vous connecter.

> Astuce : dans **Authentication → Providers → Email**, vous pouvez désactiver
> *"Enable sign-ups"* pour empêcher la création d'autres comptes. Seul le vôtre
> existera.

---

## Étape 4 — Récupérer vos clés et les coller dans le site

1. Menu de gauche : **Project Settings** (roue dentée) → **API**.
2. Repérer :
   - **Project URL** → ressemble à `https://abcd1234.supabase.co`
   - **Project API keys → `anon` `public`** → longue chaîne commençant par `eyJ…`
3. Ouvrir le fichier **`supabase-config.js`** du site et remplacer :

```js
const SUPABASE_URL = "https://abcd1234.supabase.co";   // votre Project URL
const SUPABASE_ANON_KEY = "eyJhbGciOi...votre_cle...";  // votre clé anon public
```

> ⚠️ Utilisez bien la clé **anon public**, jamais la clé `service_role`.
> La clé anon est conçue pour être publique : aucun risque à la mettre ici,
> la sécurité vient des règles RLS de l'étape 2.

4. Enregistrer, puis envoyer la modification sur GitHub (Vercel republie tout seul).

---

## Étape 5 — Utiliser l'admin

1. Aller sur `votre-site.com/admin.html`.
2. Se connecter avec l'email + mot de passe de l'étape 3.
3. Remplir le formulaire (titre, lien, description, URL d'image) → **Ajouter**.
4. La réalisation apparaît aussitôt sur `realisations.html` pour tous les visiteurs.
5. Pour retirer un projet : bouton **Supprimer** dans la liste.

---

## À propos des images

Le site attend une **URL d'image** (lien direct vers une photo déjà en ligne).
Options simples pour obtenir une URL :
- Une capture de votre projet hébergée sur un service gratuit (ex. **ImgBB**,
  **Cloudinary**, ou même le **Storage** de Supabase).
- L'URL d'une image déjà publiée sur le site du client.

Pour utiliser le Storage de Supabase : **Storage** → **New bucket** (nom `works`,
cocher *Public*) → téléverser l'image → copier son lien public → le coller dans
le champ "URL de l'image".

---

## La page admin est-elle protégée ?

- Elle est exclue des moteurs de recherche (`noindex`).
- Surtout, **aucune action n'est possible sans connexion Supabase** : un visiteur
  qui tombe sur `admin.html` verra seulement un écran de connexion, et même en
  lisant le code il ne pourra rien écrire dans la base (règles RLS).
- Gardez simplement l'URL `admin.html` et vos identifiants pour vous.
